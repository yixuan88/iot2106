04   Latency Measurement Chain

Triggered by M5StickC long-press A. Each hop is measured independently; LoRa RTT is derived mathematically.

①
BLE RTT
M5StickC sends PING, Pi echoes PONG, M5 computes RTT → sends BLRTT:<ms>
②
Serial RTT
Admin metadata request to local ESP32 over USB — pure serial, no radio
③
Mesh RTT
MESHPING over LoRa, remote gateway echoes MESHPONG. 20s timeout + retry at 10s
④
LoRa RTT
Derived: mesh_rtt − 2 × avg_serial_rtt (removes USB overhead from both ends)
⑤
WiFi RTT
Browser GET /api/ping — measured client-side, includes HTTP round-trip
Path comparison shown in web UI:   WiFi path = WiFi RTT + Serial RTT + LoRa RTT + Serial RTT   |  BLE path = BLE RTT + Serial RTT + LoRa RTT + Serial RTT

06   MQTT Topic Map
mesh/ ├── topic/<name> → Group chat · bidirectional WiFi ↔ LoRa · {v, from, text, zone, rssi, snr, hops} ├── dm/<username> → Direct message · routed via LoRa if recipient not local · {v, from, to_user, text} ├── presence/<user> → Online / offline · retained · {status, username, ts} ├── file/notify/<id> → File transfer complete · retained · {transfer_id, filename, size, from} ├── gateway/<id>/status→ Heartbeat every 5s · retained · {online, ble{}, mesh{}, wifi_users, zone} ├── latency/progress → Step-by-step measurement updates · {step, status, value, results{}} ├── ack/sent → msg_id assigned, sent over LoRa · {msg_id, from, ts} └── ack/delivered → ACK received from remote gateway · {msg_id, ts}

09   Graceful Degradation & Resilience
Failure ConditionSystem Behaviour
No LoRa32 connected Gateway starts normally; BLE and WiFi/MQTT still work for local communication
Remote gateway offline LWT marks it offline instantly; topology removes it after 150s of no status
LoRa channel congestion _lora_paused flag suppresses 60s status broadcasts during mesh ping measurements
Message not ACK'd Pending ACKs expire after 120s; pruned every 5s in the status loop
File transfer chunk loss Per-chunk CRC32 validation; META_SEQ packet carries filename + sender; max 50 KB
BLE client disconnects Reconnection count and average time-to-reconnect tracked and shown in web UI
MQTT broker disconnect Last Will & Testament marks gateway offline; paho-mqtt auto-reconnects
